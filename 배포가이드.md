# Part 1: GitHub에 코드 올리기

## 1-1. GitHub 저장소 생성

1. [github.com](https://github.com) 로그인
2. 우측 상단 **+** → **New repository**
3. 입력:
   - Repository name: `neowana` (또는 원하는 이름)
   - Public 선택
   - **Create repository** 클릭

## 1-2. 코드 업로드 (Git 없이)

1. 저장소 페이지에서 **uploading an existing file** 클릭
2. `랜챗` 폴더 안의 **server** 폴더와 **neowana** 폴더 내용을 드래그해서 올리기
   - 또는 `server` 폴더 전체, `neowana` 폴더 전체를 각각 업로드
3. **Commit changes** 클릭

### (선택) Git으로 올리기

```bash
cd C:\Users\USER-PC\Desktop\랜챗
git init
git add .
git commit -m "initial"
git branch -M main
git remote add origin https://github.com/당신아이디/neowana.git
git push -u origin main
```

# Part 2: Railway에 서버 배포

## 2-1. Railway 가입 및 프로젝트 생성

1. [railway.app](https://railway.app) 접속
2. **Login** → **GitHub**으로 로그인
3. **New Project** 클릭
4. **Deploy from GitHub repo** 선택
5. 방금 만든 `neowana` 저장소 선택
6. **Configure** 또는 **Deploy** 클릭

## 2-2. 서버 폴더 지정

1. 프로젝트 대시보드에서 생성된 서비스 클릭
2. **Settings** 탭
3. **Root Directory** 또는 **Source**에서:
   - `server` 입력 (저장소에 server 폴더가 있을 때)
   - 또는 `neowana/server` 등 실제 경로에 맞게 입력
4. **Build Command**: `npm install` (기본)
5. **Start Command**: `npm start` (기본)

## 2-3. 환경 변수 및 도메인

1. **Settings** → **Networking** → **Generate Domain** 클릭
2. 생성된 URL 복사 (예: `https://neowana-production.up.railway.app`)
3. 이 URL이 **서버 주소**입니다.

## 2-4. (저장소 구조에 따라) server만 별도 배포

저장소 루트에 server가 없다면:

1. **New Project** → **Empty Project**
2. **Add Service** → **GitHub Repo**
3. 저장소 선택 후 **Root Directory**에 `server` 입력

# Part 3: 앱 config 수정

## 3-1. config.dart 수정

`랜챗\neowana\lib\config.dart` 파일을 열고:

```dart
import 'package:flutter/foundation.dart';

String get serverUrl {
  if (kIsWeb) {
    return 'https://여기에-Railway에서-받은-URL';  // 마지막 / 제외
  }
  return 'https://여기에-Railway에서-받은-URL';
}
```

예시:
```dart
return 'https://neowana-production.up.railway.app';
```

## 3-2. Railway URL에 /ws 경로 없이 입력

- ✅ `https://xxx.railway.app`
- ❌ `https://xxx.railway.app/ws` (X)

앱 코드에서 이미 `/ws`를 붙이므로 base URL만 넣습니다.

# Part 4: Vercel에 웹 앱 배포

## 4-1. Vercel 가입 및 프로젝트

1. [vercel.com](https://vercel.com) 접속
2. **Sign Up** → **Continue with GitHub**
3. **Add New** → **Project**
4. GitHub 저장소 목록에서 `neowana` 선택
5. **Import** 클릭

## 4-2. Build 설정

1. **Root Directory** 옆 **Edit** 클릭
2. `neowana` 입력 (neowana 폴더가 루트에 있다면)
3. **Framework Preset**: Other
4. **Build Command**:
   ```bash
   flutter build web
   ```
5. **Output Directory**:
   ```bash
   build/web
   ```

## 4-3. Flutter 환경 설정 (Vercel)

Vercel에서 Flutter를 쓰려면 추가 설정이 필요할 수 있습니다.

**방법 A: vercel.json 추가**

`neowana` 폴더에 `vercel.json` 파일 생성:

```json
{
  "buildCommand": "flutter pub get && flutter build web",
  "outputDirectory": "build/web",
  "installCommand": "curl -o flutter.tar.xz https://storage.googleapis.com/flutter_infra_release/releases/stable/windows/flutter_windows_3.19.0-stable.zip || true"
}
```

**방법 B: GitHub Actions로 빌드 (추천)**

Flutter 빌드는 복잡하므로, **로컬에서 빌드 후 업로드**하는 편이 더 쉽습니다:

### 로컬 빌드 후 Vercel에 업로드

1. 터미널에서:
   ```bash
   cd C:\Users\USER-PC\Desktop\랜챗\neowana
   flutter build web
   ```
2. [vercel.com](https://vercel.com) → **Add New** → **Project**
3. **Import Third-Party Git Repository** 대신 **Browse** 또는 수동 업로드
4. 또는: `neowana/build/web` 폴더를 Vercel에 드래그 앤 드롭

# 문제 해결

**"서버 연결 실패"**
- config.dart의 URL에 `https://` 포함 여부 확인
- Railway 서비스가 실제로 실행 중인지 확인
- Railway URL 마지막 `/` 제거

**"CORS 오류"**
- server의 CORS 설정 확인 (이미 `*`로 열어둠)

**"빌드 실패"**
- Flutter 버전: `flutter doctor` 확인
- `flutter pub get` 후 `flutter build web` 재시도
